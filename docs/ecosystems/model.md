# Ecosystem Model

An ecosystem is a logical grouping of:

- repositories
- conversations
- assets
- deployments
- services

## EXAMPLE: BARIBUDOS ECOSYSTEM

- baribudos-studio (control system)
- baribudos-website (public interface)
- baribudos-api (backend services)

## RULES

- Ecosystems are semantic, not physical
- Repositories can belong to multiple ecosystems
- Ecosystems define dependency direction
- Control flows downward, data flows upward
